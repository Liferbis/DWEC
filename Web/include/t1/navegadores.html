WorldWideWeb

Representa la información accesible globalmente a través de la red de internet

Componentes:

  • Hardware
    • Los componentes físicos de Internet (hubs, repetidores, pasarelas…)
  • Software
    • Los protocolos de comunicaciones (http, IP, FTP…)
    • Otros(DNS (nombres de dominio) y software especifico)

Configuración arquitectónica más habitual

  • Servidor: es un proceso proveedor de servicios
  • Cliente: Cliente es un componente consumidor de servicios
    • ¿QUE ES?
      • El componente mas habitual del cliente es el Navegador Web
      • Componente software que se utiliza en el cliente y que permite acceder al contenido ofrecido por los servidores de Internet sin la necesidad de que el usuario instale un nuevo programa
      • Aplicación, distribuida habitualmente como software libre, que permite a un usuario acceder (y, normalmente, visualizar) a un recurso publicado por un servidor Web a través de Internet y descrito mediante una dirección URL (Universal ResourceLocator
    • TIPOS DE CLIENTE:
      • Clientes ligeros o livianos:
        Que por sí solo no pueden realizar ninguna operación real, nada más que conectarse con el servidor.
      • Clientes complejos:
        Son los utilizados actualmente. Utilizan lenguajes como Java o DHTML para aumentar funcionalidades y flexibilizar al usuario. Además de conectarse al servidor son capaces de procesar o sincronizar datos.

Criterios de clasificación:

  • Plataforma de ejecución
    Es el SO donde pueden ejecutarse los navegadores
  • Características del navegador
    Son funcionalidades adicionales asociadas a la experiencia del usuario a la hora de navegar (sin instalar extensiones), como, administración de marcadores, gestores de descarga, almacenamiento de contraseñas y datos de formularios, herramientas de búsqueda, etc....
  • Personalización de la interfaz
    Son las funciones que ayudan a usuario en la navegación y la accesibilidad.. como son: navegación por pestañas, bloqueadores de ventas emergentes, la visualización de formatos de ficheros (PDF), opciones de zoom, funciones avanzadas de búsqueda, etc..
  • Soporte de tecnologías Web
    Indica los tipos de lenguajes o tecnologías que soportan o son capaces de ejecutar, estos lenguajes son, CSS, Java, lenguajes de script del cliente (JavaScript), RSS o Atom (sindicación de contenidos), XHTML (HTML con formato XML), etc..
  • Licencia de software
    Hay en el mercado navegadores de codigo libre como, Mozilla Firefox o Google Chrome. Y navegadores propietarios como Internet Explorer o Safari. Generalmente todos son gratuitos, aunque hay excepciones como OmniWeb.

Arquitectura del navegador

  • Interfaz de usuario
    Es la capa que actúa de interfaz entre el usuario y el motor del buscador. Incluye funcionalidades como la barra de direcciones, el botón de avance/retroceso, el menú de marcadores, etc. (en general, todas las partes visibles del navegador, excepto la ventana principal donde se muestra la página solicitada). En algunos casos puede comunicarse con el SO para el manejo de sesiones de usuario o el almacenamiento de preferencias de visualización y configuración.
  • Motor de búsqueda
    Coordina las acciones entre la interfaz y el motor de renderización. Su función principal es la de cargar una dirección (URL), soportar los mecanismos de navegación (ir página anterior, posterior o recarga). También gestiona las alertas de JavaScript, el proceso de carga de la página y la administración de las preferencias de ejecución del motor de renderización.
  • Motor de renderización
    es responsable de mostrar el contenido solicitado, ya que interpreta código de la pagina web. Por ejemplo, si el contenido solicitado es HTML, será el responsable de analizar el código HTML y CSS y de mostrar el contenido analizado en la pantalla. Este modulo establece las dimensiones exactas de cada elemento a mostrar y a veces se encarga de posicionarlos en la página
  • Comunicaciones o Red
    es responsable de las llamadas de red, como las solicitudes HTTP, FTP... Tiene una interfaz independiente de la plataforma y realiza implementaciones en segundo plano para cada plataforma. Se encarga de codificar los datos obtenidos en función de su tipo (texto, audio, video). A veces (según navegadores ) almacena una cache para de elementos accedidos recientemente.
  • Intérprete de JavaScript
    permite analizar y ejecutar el código JavaScript. Este modulo puede ser configurado (incluso deshabilitado) por cuestiones de seguridad. Dependiendo de las tecnologías soportadas por los navegadores puede ser que existan interpretes específicos de otros lenguajes como Applets de java ,AJAX o ActionScript
  • Parser XML
    En este módulo los navegadores suelen cargar en memoria una representación en árbol (árbol DOM) de la página para poder acceder a los diferentes elementos mas rápido por parte del navegador
  • Servidor de la interfaz o componentes de visualización
    Este módulo ofrece funcionalidades relacionadas con la visualización de los contenidos. Ofrece primitivas de dibujo y posicionamiento en una ventana permite presentar widgets básicos( como ventanas y cuadros combinados) y un conjunto de fuentes tipográficas. Utiliza métodos de la interfaz de usuario del sistema operativo en segundo plano
  • Almacenamiento de datos
    es una capa de persistencia o almacén de datos para los diferentes módulos del navegador. El navegador necesita guardar todo tipo de datos en el disco duro (por ejemplo, las cookies, o certificados de seguridad). La nueva especificación de HTML (HTML5) define el concepto de "base de datos web", que consiste en una completa (aunque ligera) base de datos del navegador
